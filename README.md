# VLESS VPN Panel

Self-hosted VPN panel for personal use or resale. Built on Xray-core with VLESS+Reality transport, managed via a web panel and Telegram bot.

## Features

- **VLESS + Reality** — traffic is indistinguishable from normal HTTPS (camouflages as Microsoft)
- **Web panel** — manage users and devices, view traffic charts, online status, connection history
- **User editor** — change password, expiry date, Telegram ID without recreating the account
- **Per-device QR codes** — scan to connect in seconds
- **Statistics** — daily traffic charts (Recharts), real-time online devices, connection log with client IPs
- **Audit log** — records logins, device creation/deletion, user management (admin only, paginated)
- **Login protection** — math CAPTCHA + IP lockout for 15 min after 3 failed attempts + Telegram alert to admin
- **Telegram bot** — users manage their own devices via inline keyboards; no password needed if pre-linked by admin
- **Setup guide** — built-in page with step-by-step HAPP/Hiddify/v2rayN setup instructions
- **CI/CD** — automatic deploy to production on every push to `main` via GitHub Actions

## Server requirements

- **OS**: Ubuntu 22.04 LTS
- **RAM**: 1 GB recommended (512 MB minimum)
- **CPU**: 1 vCPU
- **Ports**: 80 and 443 must be open and free
- **Domain**: an A record pointing to the server IP (e.g. `vpn.example.com → 1.2.3.4`)
- **Docker**: installed automatically by the setup script if missing

## Deploy to a new server

Run one command on a fresh Ubuntu server:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/pavelgla/vless-vpn/main/setup_server.sh)
```

The script will ask for domain, admin credentials and Telegram bot token, then:

1. Install Docker
2. Create a `deploy` user with Docker access
3. Configure SSH key for GitHub Actions
4. Generate a GitHub deploy key — pause and wait for you to add it to the repo
5. Clone the repo to `/opt/vless-vpn`
6. Create `.env` (DB password and JWT secret are auto-generated)
7. Run `install.sh` — generates Reality keys, obtains TLS cert, builds and starts all containers

After completion the panel is at `https://YOUR_DOMAIN`.

> See [deploy_instructions.md](deploy_instructions.md) for a full step-by-step breakdown.

## Reinstall / change domain

Running `install.sh` again on an existing installation detects `.installed` and asks whether to reinstall. To fully wipe data and start over:

```bash
bash reset.sh
```

## CI/CD — automatic deployment

After `setup_server.sh` completes, every push to `main` automatically deploys to production:

```
git push → GitHub Actions → SSH into deploy@SERVER → git pull + docker compose build + up
```

Required GitHub Secrets (set once, never change unless server changes):

| Secret | Value |
|--------|-------|
| `SERVER_HOST` | Server IP or domain |
| `SERVER_USER` | `deploy` |
| `SSH_PRIVATE_KEY` | Private key generated by `setup_server.sh` |

After moving to a new server: update `SERVER_HOST` in GitHub Secrets.

## Adding users

**Via web panel:**
1. Log in at `https://YOUR_DOMAIN` with superadmin credentials
2. Open **Users** → **Add user**
3. Set login, password, optional expiry date and Telegram ID

**Edit existing user** — click **Изменить** in the users table to change password, expiry, or Telegram ID.

**Via Telegram bot (superadmin):**
```
/adduser login password 2025-12-31
```

## Adding a device (getting a VLESS link)

Users can add their own devices from the panel (limit: 5 per user). Admins can add devices for any user from the Users section.

1. Log in → **My devices** → **Add device**, enter a name (e.g. "iPhone")
2. A QR code and VLESS link appear immediately
3. Import the QR code or link into your VLESS client

**VLESS link format:**
```
vless://UUID@domain:443?type=tcp&security=reality&pbk=PUBLIC_KEY&sid=SHORT_ID&sni=www.microsoft.com&fp=chrome&flow=xtls-rprx-vision#DeviceName
```

> **Important:** `flow=xtls-rprx-vision` is required — make sure your client has it enabled.

## Telegram bot

Users are linked by the admin setting their Telegram ID in the web panel. On first `/start` the bot auto-detects the link — no password needed.

**User commands:**
```
/start          — link Telegram account (or auto-link if admin pre-set the ID)
/devices        — list devices with inline QR and delete buttons
/adddevice name — add a new device (also available via ➕ button in /devices)
/deldevice name — remove a device with confirmation
/qr name        — get QR code for a device
/traffic        — traffic stats for the last 30 days
/expire         — show account expiry date
/help           — all commands
```

**Superadmin commands:**
```
/adduser login password [YYYY-MM-DD]
/users
/setexpire login YYYY-MM-DD
/disable login
/enable login
/stats
```

## Login security

- **CAPTCHA** — a math question is shown on every login attempt (server-generated, one-use, 5 min TTL)
- **IP lockout** — after 3 failed attempts the IP is blocked for 15 minutes
- **Telegram alert** — superadmin receives a Telegram message when an IP gets locked out

## Statistics

The API polls Xray every 30 seconds via Docker socket:

- **Traffic** — daily bytes up/down per device, stored in `traffic_daily` table
- **Online status** — via `xray api statsgetallonlineusers`
- **Client IPs** — via `xray api statsonlineiplist` and access log parsing
- **Connection log** — stored in `connection_log` table, shown per device in the panel
- **Audit log** — all admin actions (login, device CRUD, user management) in `audit_log` table, paginated

With CI/CD configured — just push to `main`. Without CI/CD:

```bash
cd /opt/vless-vpn
git pull
docker compose build --pull
docker compose up -d
```

Database migrations run automatically on API startup.

## Backup

```bash
bash scripts/backup.sh
```

Saves a gzipped PostgreSQL dump to `./backups/`. Keeps the last 30 backups.

Set up daily automatic backups:
```bash
echo "0 3 * * * root bash /opt/vless-vpn/scripts/backup.sh >> /var/log/vpn-backup.log 2>&1" \
  | sudo tee /etc/cron.d/vpn-backup
```

## Architecture

```
Internet
    |
    v :80 / :443
  nginx (TCP stream, ssl_preread)
    |
    +-- SNI = YOUR_DOMAIN --> nginx HTTPS (:8443)
    |                               |
    |                        /api/* --> api:3000 (Fastify + JWT)
    |                        /      --> frontend:80 (React SPA)
    |
    +-- any other SNI -------> xray:443 (VLESS + Reality)
                                    |
                            xray/config.json
                            (managed by API, hot-reloaded via SIGUSR1)
```

**Traffic flow for VPN clients:**
1. Client connects to `YOUR_DOMAIN:443` with SNI `www.microsoft.com`
2. Nginx routes the raw TCP stream to xray (non-domain SNI → xray)
3. Xray performs the Reality handshake — traffic looks like HTTPS to Microsoft
4. VLESS layer authenticates the client by UUID + xtls-rprx-vision flow
5. Traffic is forwarded to the internet from the server

> **Note:** Do not use `proxy_protocol` between nginx and xray — it is incompatible with REALITY and causes silent connection failures.

## Security notes

- Secrets live in `.env` only (never committed to git)
- `xray/config.json` is in `.gitignore` — contains live user UUIDs
- JWT tokens are blacklisted on logout
- Login page has CAPTCHA and IP-based brute-force lockout
- Superadmin cannot be deleted via API
- Reality protocol makes VPN traffic indistinguishable from normal HTTPS
- All admin actions are recorded in the audit log with timestamps and IPs
- CI/CD uses a read-only deploy key for GitHub and a separate key for server access
